# Mongoose Models and Schemas

## Reflections

1: It makes the codebase easier to understand, easier and safer to change, and easier to scale as the project grows.

2: PUT: replace the entire resource, request body contains a full representation. PATCH: partially update a resource; the body usually includes only the fields you want to change.
   If PUT /:id endpoint requires all fields and overwrites missing fields, it more closely resembles PATCH
   
3: A small and fast response would be nice, so the user can understand that the delete is done.
